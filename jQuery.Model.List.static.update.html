update

  • function
 

Update is used to update a set of model instances on the server. By implementing update along with the rest of the [jquery.model.services service api], your models provide an abstract API for services.

static.update(ids, attrs, success, error)

Update is used to update a set of model instances on the server. By implementing update along with the rest of the [jquery.model.services service api], your models provide an abstract API for services.

Parameters

  1. ids {Array}

    the ids of the model instance

  2. attrs {Object}

    Attributes on the model instance

  3. success {function()}

    the callback function. It optionally accepts an object of attribute / value pairs of property changes the client doesn't already know about. For example, when you update a name property, the server might update other properties as well (such as updatedAt). The server should send these properties as the response to updates. Passing them to success will update the model instances with these properties.

  4. error {function()}

    a function to callback if something goes wrong.

The easist way to implement update is to just give it the url to put data to:

$.Model.List("Recipe",{
  update: "PUT /thing/update/"
},{})

Or you can implement update manually like:

$.Model.List("Thing",{
  update : function(ids, attrs, success, error){
       return $.ajax({
          url: "/thing/update/",
          success: success,
          type: "PUT",
          data: { ids: ids, attrs : attrs }
          error: error
       });
  }
})

Then you update models by calling the prototype update method.

listInstance.update({ name: "Food" })

By default, the request will PUT an array of ids to be updated and the changed attributes of the model instances in the body of the Ajax request.

{ 
    ids: [5,10,20],
    attrs: { 
        name: "Food" 
    } 
}

Your server should send back an object with any new attributes the model should have. For example if your server udpates the "updatedAt" property, it should send back something like:

// PUT /recipes/4,25,20 { name: "Food" } ->
{
  updatedAt : "10-20-2011"
}
